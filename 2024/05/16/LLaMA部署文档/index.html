概述

模型微调方法

全模型微调 (Full Model Fine-Tuning)

​ 这种方法涉及对预训练模型中的所有参数进行微调。这通常可以实现最好的性能，但需要大量计算资源和较大的数据集。适用于在目标任务中有丰富数据的情况

冻结层 (Layer Freezing)

​ 冻结预训练模型的大部分层，仅微调特定的几层（通常是顶部几层）。这种方法可以显著减少计算成本，适用于数据量较少的情况。

Low-Rank Adaptation (LoRA)

​ LoRA是一种参数高效的微调方法，旨在减少在微调大规模预训练模型时所需的计算资源和存储需求。LoRA 通过对预训练模型中的特定参数进行低秩分解，并仅微调这些分解后的低秩参数，而不是对模型的全部参数进行调整。

环境准备

系统环境

  • Ubuntu Server 22.04 LTS
  • RAM 256GB
  • VRAM 80GB

运行环境

  • conda(不是必须)
  • python 3.10
  • torch+cuda

部署过程

在你的conda环境中

部署frcp

  1. Download this file: https://cdn-media.huggingface.co/frpc-gradio-0.2/frpc_linux_amd64
  2. Rename the downloaded file to: frpc_linux_amd64_v0.2
  3. Move the file to this location: PATH_TO_Python/site-packages/gradio

yaml文件解析

Model Configuration

  • model_name_or_path

    • 指定了使用的基础模型，即Meta-Llama-3-8B-Instruct。
Training Method

  • stage:

    • 表示进行微调（Supervised Fine-Tuning）。

  • do_train:

    • 表示进行训练。

  • finetuning_type:

    • 指定了使用LoRA方法进行微调。

  • lora_target:

    • 指定了要微调的模型层，具体是q_proj和v_proj层。
Distributed Data Parallel (DDP)

  • ddp_timeout:

    • 设置DDP的超时时间。
Dataset Configuration

  • dataset:

    • 指定了要使用的两个数据集，分别是identity和alpaca_gpt4_en。

  • template:

    • 使用的模板，llama3。

  • cutoff_len:

    • 设置输入序列的最大长度为1024。

  • max_samples:

    • 使用的最大样本数量为1000。

  • overwrite_cache:

    • 表示覆盖缓存。

  • preprocessing_num_workers:

    • 指定了预处理数据时使用的工作线程数量为16。
Output Configuration

  • output_dir:

    • 指定了训练输出的目录。

  • logging_steps:

    • 每10步记录一次日志。

  • save_steps:

    • 每500步保存一次模型。

  • plot_loss:

    • 表示绘制损失图。

  • overwrite_output_dir:

    • 表示覆盖输出目录。
Training Configuration

  • per_device_train_batch_size:

    • 每个设备的训练批次大小为1。

  • gradient_accumulation_steps:

    • 梯度累积的步数为2，即每2步更新一次梯度。

  • learning_rate:

    • 学习率设置为0.0001。

  • num_train_epochs:

    • 训练的轮数为3轮。

  • lr_scheduler_type:

    • 学习率调度器类型为余弦调度。

  • warmup_steps:

    • 预热步数为总训练步数的10%。

  • fp16:

    • 使用16位浮点数进行训练。
Evaluation Configuration

  • val_size:

    • 验证集的大小为总数据集的10%。

  • per_device_eval_batch_size:

    • 每个设备的验证批次大小为1。

  • evaluation_strategy:

    • 使用逐步评估策略。

  • eval_steps:

    • 每500步进行一次评估。
